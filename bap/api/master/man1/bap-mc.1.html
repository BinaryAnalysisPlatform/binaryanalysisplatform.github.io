NAME

bap-mc - BAP machine instruction playground  

SYNOPSIS


 bap-mc [PLUGIN OPTION]... --list-formats
 bap-mc [OPTION]... DATA
 

DESCRIPTION

Disassemble a string of bytes. This is the BAP machine code playground. It is intended to mimic a subset of llvm-mc functionality using the BAP disassembly backend.  

ARGUMENTS

DATA
String to disassemble. If not specified read stdin
 

OPTIONS

--list-formats=TYPE
Print available formats for the given data TYPE. The TYPE must be one of `insn', `inst', `bil' or `bir'
--addr=VAL (absent=0x0)
Specify an address of first byte
--arch=ARCH (absent=x86_64)
Target architecture
--help[=FMT] (default=auto)
Show this help in format FMT. The value FMT must be one of `auto', `pager', `groff' or `plain'. With `auto', the format is `pager` or `plain' whenever the TERM env var is `dumb' or undefined.
--only-one
Stop after the first instruction is decoded
--show-bil[=VAL] (default=pretty)
Output BIL code. Optional value specifies format and can be sexp, marshal, bin, adt, pretty. Defaults to `pretty`, i.e., in a BIL concrete syntax
--show-bir[=VAL] (default=pretty)
Output for each instruction in particular. Accepted values are sexp, marshal, bin, pretty
--show-inst[=VAL], --show-insn[=VAL] (default=pretty)
Print instructions, using specified format . can be sexp, marshal, bin, adt, asm, pretty. Defaults to `asm'.
--show-kinds
Output instruction kinds.
--show-size
Output recognized instruction length
--version
Show version information.
 

PLUGIN OPTIONS

-l PATH
Dynamically loads file PATH.plugin. A plugin must be compiled with bapbuild tool using bapbuild PATH.plugin command.
-LPATH
Add PATH to a set of search paths. Plugins found in the search paths will be loaded automatically.
--list-plugins
List all available plugins or list plugins that provide some features, e.g. --list-plugins=disassembler,lifter
--list-tags
List all available plugin tags
--disable-plugin
Don't load PLUGIN automatically
--disable-autoload
Disable auto loading of plugins
--no-PLUGIN
Don't load PLUGIN automatically
 

EXAMPLES

The following hex representations are supported: 


         0x31 0xd2 0x48 0xf7 0xf3
         \x31\xd2\x48\xf7\xf3
         31 d2 48 f7 f3
         31d248f7f3
INPUT: Supplied via stdin or on the command-line
echo "0x31 0xd2 0x48 0xf7 0xf3" | bap-mc --show-inst --show-bil
