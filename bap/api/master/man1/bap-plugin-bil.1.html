NAME

bil - Provides bil optimizations  

SYNOPSIS

bil [OPTION]...  

DESCRIPTION

Controls the BIL transformation pipeline.

The BIL transformation pipeline is used to transform the BIL code of each machine instruction and is applied to the output of a lifter. Each piece of transformation is called a pass. This plugin enables full control over the pass selection as well as provides some predefined passes. See the --bil-list-passes for a comprehensive list of available transformations. The pipeline is formed as a concatenation of all normalization passes that correspond to the selected level of normalization (setting it to zero will disable normalizations), all optimization passes applicable to the selected level of optimization (again the zero level corresponds to an empty list of optimizations), and all passes selected via the --bil-passes command line argument. All passes are applied until a fixed point is reached (i.e., until the outcome of the pipeline is equal to the income) or until a loop is detected. Thus to prevent infinite looping each registered pass should be a pure function of its inputs (same input shall result in the same output).

A new pass could be registered via the Bil.register_pass function. It won't be automatically selected, so it should be specified explicitly via the --bil-passes command line argument.  

OPTIONS

--help[=FMT] (default=auto)
Show this help in format FMT. The value FMT must be one of `auto', `pager', `groff' or `plain'. With `auto', the format is `pager` or `plain' whenever the TERM env var is `dumb' or undefined.
--list-passes
List all available passes and exit
--normalization=VAL (absent=1)
Specifies the BIL normalization level. The normalization process doesn't change the semantics of a BIL program, but applies some transformations to simplify it. There are two BIL normal forms (bnf): bnf1 and bnf2, both of which are described in details in bap(3). Briefly, bnf1 produce the BIL code with load expressions that applied to a memory only. And bnf2 also adds some more restrictions like absence of let-expressions and makes load/store operations sizes equal to one byte. So, there are next possible options for normalization level: 0 - disables normalization; 1 - produce BIL in bnf1; 2 - produce BIL in bnf2
--optimization=VAL (absent=1)
Specifies the optimization level. Level 0 disables optimization, and level 1 performs regular program simplifications, i.e., applies constant folding, propagation, and elimination of dead temporary (aka virtual) variables.
--passes=VAL
Selects the list and the order of analyses to be applied during the lifing to BIL code.
