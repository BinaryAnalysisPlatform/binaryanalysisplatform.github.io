Module Bap.Std.Bil

module Bil: sig .. end
Main BIL module

This module defines BIL language and is useful to write BIL programs and expressions.

Example: 

Bil.([
          v := exp_of_op src lsr int32 1;
          r := exp_of_op src;
          s := int32 31;
          while_ (var v <> int32 0) [
            r := var r lsl int32 1;
            r := var r lor (var v land int32 1);
            v := var v lsr int32 1;
            s := var s - int32 1;
          ];
          dest := var r lsl var s;
        ]) cond

iter visitor bil apply a visitor for each node of a BIL forest.

val is_referenced : Bap.Std.var -> stmt list -> bool
is_referenced x p is true if x is referenced in some expression or statement in program p: BUG: currently occurrences on the left hand side of the assignment operator is considered a reference too.
val is_assigned : ?strict:bool -> Bap.Std.var -> stmt list -> bool
is_assigned x p is true if there exists such Move statement, that x occures on the left side of it. If strict is true, then only unconditional assignments. By default, strict is false
val prune_unreferenced : stmt list -> stmt list
prune_unreferenced p remove all assignments to variables that are not used in the program p
val normalize_negatives : stmt list -> stmt list
normalize_negatives p transform x + y to x - abs(y) if y < 0
val substitute : exp -> exp -> stmt list -> stmt list
substitute x y p substitutes each occurrence of expression x by expression y in program p
val substitute_var : Bap.Std.var -> exp -> stmt list -> stmt list
substitute_var x y p substitutes all occurences of variable x by expression y
val fold_consts : stmt list -> stmt list
fold_consts evaluate constant expressions. Note: this function performs only one step, and has no loops, it is supposed to be run using a fixpoint combinator.
class constant_folder : mapper
constant_folder is a class that implements the fold_consts 
val fixpoint : (stmt list -> stmt list) -> stmt list -> stmt list
fixpoint f applies transformation f until fixpoint is reached. If the transformation orbit contains non-trivial cycles, then the transformation will stop at an arbitrary point of a cycle.
