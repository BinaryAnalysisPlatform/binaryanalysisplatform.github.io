Module Bap_c_type_mapper_intf

module Bap_c_type_mapper_intf: sig .. end
Type mapper, visitor, iterator, finder all in one.

The interface is wrapped into a monad, that allows, by choosing a proper monad implement all the above morphisms and iterators.

Each syntactical element t:T of the type system is represented with three methods:

  • enter_T t
  • map_T t
  • leave_T t
The map_T t method first calls enter_T t, the applies a deep mapping of the t to t' and finally calls leave_T t'.

Override enter_T if an element shouldn't be morphed. The combination of enter_t, leave_T allows to perform different visiting strategies. If mapping is needed then a map_T method should be overriden. A usual pattern would be: 

      class my_mapper = object(self)
        inherit base as super

        method map_T t =
          super#map_T t >>| self#my_transformation

        method private my_transformation t = t
      end
