module Disasm: sig .. end
Disassembled program
val insns : Bap_disasm.t -> (Image_internal_std.mem * Bap_disasm.insn) Bap_types.Std.seq
returns all instructions that was successfully decoded in an ascending order of their addresses. Each instruction is accompanied with its block of memory.
val blocks : Bap_disasm.t -> Bap_disasm.block Image_internal_std.table
returns a mapping from memory regions to basic blocks
val insn_at_mem : Bap_disasm.t -> Image_internal_std.mem -> Bap_disasm.insn option
performs fast lookup for an instruction that occupies exactly the given piece of memory. If you need to find all instructions that lies in a given region of memory, use insns_of_mem or insns_of_block functions.
val insns_at_mem : Bap_disasm.t ->
       Image_internal_std.mem ->
       (Image_internal_std.mem * Bap_disasm.insn) Bap_types.Std.seq
insns_of_mem returns all instructions that occupies memory regions that have intersections with mem.
val mems_of_insn : Bap_disasm.t -> Bap_disasm.insn -> Image_internal_std.mem Bap_types.Std.seq
returns a sequence of memory regions occupied by the given instruction.
val insn_at_addr : Bap_disasm.t ->
       Bap_types.Std.addr -> (Image_internal_std.mem * Bap_disasm.insn) option
insn_at_addr t addr finds instruction to which the addr belongs. In other words if instruction at given address is found it doesn't mean, that it starts at this address. Consider comparison with min_addr if you need to match starting address only.
val span : Bap_disasm.t -> Image_internal_std.mem Bap_types.Std.seq
returns a blocks of memory that was visited during the disassembly. The regions are merged with Memory.merge if possible. So it returns the least possible amount of contiguous memory regions
type error = [ `Failed of Core_kernel.Std.Error.t

val errors : Bap_disasm.t ->
       (Image_internal_std.mem * error) Bap_types.Std.seq
returns a list of all errors and warnings that occurred during the disassembling
