NAME

trace - manage execution traces  

SYNOPSIS


        bap --trace-dump=URI
        bap BINARY --trace-load=URI...
 

DESCRIPTION

Loads and prints traces. The plugin can be used in two modes. When called as --trace-dump it will just dump the specified trace and exit. In the second mode, it will load specified traces, so that they can be used by analysis. The loaded traces must be runs of the analyzed BINARY. The loaded traces are accessible via the Traces of the traces library.  

OPTIONS

--dump=URI
Dump a trace specified by URI
--help[=FMT] (default=auto)
Show this help in format FMT. The value FMT must be one of `auto', `pager', `groff' or `plain'. With `auto', the format is `pager` or `plain' whenever the TERM env var is `dumb' or undefined.
--load=URI
Load trace from the specified URI. The option maybe used many times to load several traces
