NAME

bapbundle - manages BAP bundles.  

SYNOPSIS

bapbundle -help
bapbundle COMMAND -help
bapbundle install [OPTION]...  BUNDLE
bapbundle remove  [OPTION]...  BUNDLE
bapbundle show    [OPTION]...  BUNDLE
bapbundle update  [OPTION]...  BUNDLE
bapbundle pack    [OPTION]...  BUNDLE

 

DESCRIPTION

A bundle is an opaque package containing anything that is requested for a program to work in a standalone mode. The bapbundle tool allows a user to create bundles, update meta information and pack resources. It also can install and remove bundles into the designated bundle repository, where they can be found by a main BAP program without extra options.

Usually the bapbundle command is used implicitly by the bapbuild, so there is no need to use the pack command directly. A user can, however, add some resource files or update meta information, using the update command.

The install command copies a BUNDLE to a known destination, that is seearched by a main program, when plugins are loaded automatically. The default destination is specified at program configuration time and usually works out of box. The destination can be overridden in the runtime, however.

The remove command removes a previously installed BUNDLE from a given destination.

 

ARGUMENTS

BUNDLE
Name of a bundle.

 

COMMON OPTIONS

-help, --help
Prints a brief summary of command line options.

 

COMMON META OPTIONS

The following options sets or updates meta information stored in the BUNDLE.

-author NAME

Set author's name. If unspecified, then a value of USER environment variable is used. Ideally, the name should be of the form Full Name [Short name] <email>

-name NAME

Sets the name of the BUNDLE to the NAME. If not specified, then a basename without an extension of the NAME will be used.

-desc TEXT
Sets the TEXT as a package description. The text should consist of a one line short description, followed by an optional detailed multiline description. The detailed description should be separated from the short one by a new line symbol.

 

INSTALL AND REMOVE OPTIONS

-destdir DIR
Specifies a directory for install and remove commands. If unspecified, then a default directory specified at configuration time will be used.

 

SHOW OPTIONS

-mFR

Shows meta information.

-rFR

Prints a list of resource files stored in the BUNDLE.

-lFR

Prints a list of libraries and other loadable code, stored in the BUNDLE.

 

UPDATE OPTIONS

Other then COMMON META OPTIONS the update command accepts the following options:

-add-resources RESOURCE-LIST

Adds resources specified by the RESOURCE-LIST to the BUNDLE. The RESOURCE-LIST is a comma separated list of resources, where each resources is either a file path or a named filepath of a form <name>=<path>, where <name> defines how a resource will accessed in the bundle. If not specified, then the <name> is equal to <path>. The RESOURCE-LIST mustn't contain spaces, unless a filename contains it. In that case the resource list must be enclosed in parenthesis.

The RESOURCE-LIST grammar: 

resources ::= resource [, resources]
resource  ::= path | name=path

 

PACK OPTIONS

Other then COMMON META OPTIONS the update command accepts the following options:

-main NAME

Set the entry point to NAME. The entry point is a module that is loaded by a bap main program.

-requires FEATURE-LIST

Define package requirements. The requirements are libraries used by the package. The FEATURE-LIST must be a comma separated list, of requirements. Each requirement is a library name, optionally followed by filename of the implementation. If implementation is provided, then it will be included in the BUNDLE. Othrewise, only the name will be stored, and the implementation will be searched at the time of loading using ocaml-findlib.

The FEATURE-LIST grammar: 

feature-list ::= feature [, feature]
feature      ::= name | name=path

-provides FEATURE-LIST

Specify a list of features provided by the package. The FEATURE-LIST should a comma separated list of names, where each name is an interface name provided by a plugin.

-resources ESOURCES

Specify a list of resources used by the package. See -add-resource option for more information.

 

EXIT STATUS

Program returns:
   - 0 if everything is ok.
   - 1 for malformed command line
   - 2 for all other errors.

 

ENVIRONMENT

Program reads USER variable to get a user name.
