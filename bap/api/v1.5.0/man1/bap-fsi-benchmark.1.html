NAME

bap-fsi-benchmark - function start identification benchmark game  

SYNOPSIS

bap-fsi-benchmark [OPTION]... binary tool ground-truth  

DESCRIPTION

Compares function start identification algorithms to the ground truth. The latter should be provided by a user.  

PLUGIN OPTIONS

-l PATH
Dynamically loads file PATH.plugin. A plugin must be compiled with bapbuild tool using bapbuild PATH.plugin command.
-LPATH
Add PATH to a set of search paths. Plugins found in the search paths will be loaded automatically.
--list-plugins
List all available plugins or list plugins that provide some features, e.g. --list-plugins=disassembler,lifter
--list-tags
List all available plugin tags
--disable-plugin
Don't load PLUGIN automatically
--disable-autoload
Disable auto loading of plugins
--no-PLUGIN
Don't load PLUGIN automatically
 

ARGUMENTS

binary (required)
The testing stripped binary.
ground-truth (required)
The ground truth. The ground truth can be an unstripped binary, or a .scm file with symbol information. In .scm file, each symbol should be in format of: (<symbol name> <symbol start address> <symbol end address>), e.g. (malloc@@GLIBC_2.4 0x11034 0x11038)
tool (required)
The tool of the function start result that we are going to evaluate. Possible values: one of `ida', `byteweight' or `internal'.
 

OPTIONS

-c VAL, --with-metrics=VAL
Print metrics. User can choose to print -cprecision, -crecall, -cF_measure, -cTP, -cFP and -cFN.
--help[=FMT] (default=auto)
Show this help in format FMT. The value FMT must be one of `auto', `pager', `groff' or `plain'. With `auto', the format is `pager` or `plain' whenever the TERM env var is `dumb' or undefined.
