NAME

primus-promiscuous - enables the promiscuous mode of execution  

SYNOPSIS

primus-promiscuous [OPTION]...  

DESCRIPTION

When this mode is enabled the Primus Machine will venture into paths with unsatisfied constraints. Basically, it means that on every branch the state is duplicated.

The program will be translated into the Trivial Condition Form, where each compound condition expression is trivialized to a variable, that is bound earlier in the block.  

OPTIONS

--help[=FMT] (default=pager)
Show this help in format FMT (pager, plain or groff).
--mode
Enable the mode.
