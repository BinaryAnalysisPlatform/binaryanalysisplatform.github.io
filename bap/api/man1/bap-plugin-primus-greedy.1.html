primus-greedy

Section: BAP Programmer's Manual (3)
Updated:
Index Return to Main Contents
 

DESCRIPTION

The greedy scheduling strategy will continue with the same state, unless the machine reaches a termination state, i.e., when the next value in a context becomes None. In that case another alive state that has a next value that is not None is chosen. If such state doesn't exist, then the Machine finally terminates. Thus this strategy will perform a depth-first traversal of the state tree, and guarantees that all paths are explored. The greedy scheduler will attempt to reschedule every time a basic block is evaluated.
