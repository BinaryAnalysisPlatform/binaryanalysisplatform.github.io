Binary Analysis Platform API.

Overview

BAP has a layered architecture currently consisting of four layers:

  • Core types - defines Binary Intermidiate Language (BIL) as well as some common basic types, commonly usable for binary analysis.
  • Memory model - loads and represents binary in the memory;
  • Disassembling - the first pass of the analysis - disassembles and lifts binary to BIL.
  • Semantic analysis - the seconds pass of decompilation, that performs lifting to IR and provides useful analysis.

How to use the library

You should start any code relying on bap-types library with a

open Bap.Std

It is a good idea, to open Core_kenel.Std before.
